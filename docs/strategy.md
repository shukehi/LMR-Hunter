# 策略文档

## 1. 文档目的

本文档定义 LMR-Hunter 的策略边界、核心命题、交易假设、信号逻辑和参数体系。

本文件回答的是“为什么做、何时做、何时不做”。

### 第一性原理定义与推导链

本项目中的“第一性原理”不是泛指“从零思考”，而是指策略成立所依赖的最小可证伪因果链。

定义冻结如下（基于真实市场微观结构重构）：

1. 强制平仓会产生非自愿、对价格完全不敏感的市价单（Taker Sell）。
2. **流动性的物理学**：价格跌幅不取决于绝对的抛压金额，而取决于**抛压金额与当前盘口承接厚度（Limit Buy Depth）的比例**。当瞬时市价抛压远大于买盘深度时，做市商为规避单边风险撤单，形成真实的“流动性真空”。
3. **回归的唯一动力**：真空砸出的深坑不会因为“历史平均价（如 VWAP）在上面”就自动反弹。真正的修复动力来自**期现套利者（Arbitrageurs）**。当永续价格被砸到显著低于现货指数价格（Index Price）时，套利者买入永续、卖出现货，赚取价差收敛利润，从而填补真空。
4. 只有当**永续-现货负基差（Basis）**深度足以覆盖套利者的手续费、资金费率风险，以及我们自身的滑点和止损成本时，它才是可交易的 edge。
5. 若以上任一条件无法被数据验证，系统只能停留在观察与研究阶段，不能推进参数冻结或实盘。

本定义是策略文档、优化计划、风控门槛和研究报告的上游约束。术语说明见 [术语表](glossary.md)。

## 2. 策略概述

### 2.1 策略名称

`清算驱动的基差回归策略 (Liquidation-Driven Basis Reversion)`

### 2.2 策略目标

识别 BTC 永续合约中由大规模强制平仓引发的短期流动性真空，通过监测**真实的盘口枯竭**和**极端的期现负基差**，在卖压衰竭时通过被动限价单参与基差修复。

### 2.3 策略立场

该策略不是趋势预测模型，不基于形态、滞后指标（如均线、VWAP、MACD）或主观盘感。

它只关心三个硬核物理事实：

1. 真实的抛压相对盘口深度有多大？
2. 永续价格相对于现货指数偏离了多少？
3. 市场是否出现了主动买盘（套利者进场）的迹象？

## 3. 使用场景与约束

本策略的初始设计假设面向以下约束：

- 账户规模较小（~1000 USD）
- 具备编程能力
- 部署于东京 VPS，端到端延迟约 `2-50ms`
- 不与 HFT 做抢单竞争，但利用低延迟抢占更优的被动挂单位置并实现精确止损
- 接受低胜率、高赔率
- 日常可投入时间有限

## 4. 策略命题

策略基于以下可检验命题：

1. 杠杆头寸被强制平仓时，会产生非自愿的市价抛售。
2. 永续合约的极端抛售会导致其价格短暂脱离现货市场（Spot），形成显著的负基差（Negative Basis）。
3. 市场中存在充足的自动化套利资本，只要基差空间大于套利无风险成本，套利者必定进场做多永续，驱动基差向 0 收敛。
4. 个人小资金可以通过极低延迟的被动挂单，在套利者大规模进场前或进场初期的极低位置提供流动性，赚取这部分基差修复的红利。

### 4.1 修复失败的条件

修复不会发生的情况：

- **现货共振下跌**：价格下跌原因不是永续杠杆踩踏，而是现货市场真实的巨量抛售。此时现货与永续同步下跌，基差未拉开，**没有“错价”可修复**。
- **连环清算未结束**：第一波强平触发了更多止损，强平引擎在脉冲式工作，你以为的“衰减”只是引擎计算下一个破产价的“网络卡顿间隔”。
- **市场结构性转变**：极端恐慌（如交易所暴雷），套利者拔网线拒绝提供流动性。

## 5. 交易假设

项目当前采用以下交易假设：

- 交易所提供稳定且可用的强平事件流、实时 Orderbook 深度流、以及现货指数价格流。
- BTC 永续合约长期存在高杠杆交易者。
- 小账户的深度挂单不会引起明显冲击。

## 6. 失效前提

以下情况会直接削弱或破坏策略前提：

- 强平发生时，现货指数同步暴跌，基差未显著扩大。
- 盘口真实承接极厚，大规模强平无法砸出足够的基差空间。
- 自动化失效检测指标持续报警。

### 6.1 自动化失效检测

| 失效指标 | 计算方式 | 阈值（首版） | 动作 |
|----------|----------|-------------|------|
| 滚动胜率崩塌 | 最近 20 笔成交的胜率 | < 30% | 自动暂停，进入 `HALTED` |
| 基差修复失败率 | 成交后基差在 3 分钟内未收敛的比例 | > 50%（最近10笔）| 告警 + 人工评估 |
| 回弹延迟趋势恶化 | 最近 7 天 vs 前 7 天的平均回弹启动延迟 | 增加 > 100% | 告警 + 人工评估 |
| 影子-实盘偏差 | 影子模式预测盈亏 vs 实盘实际盈亏的相关性 | 相关系数 < 0.5 | 自动暂停 |

## 7. 信号结构

系统信号重构为四层，全部围绕市场微观结构展开：

- `事件层`、`流动性层`、`基差层` 和 `微观衰减层` 是 MVP 硬门槛。
- 所有下单请求都必须先经过预交易风控审核。

### 7.1 事件层（The Trigger）

判断是否出现异常强平事件。
- 监听 Binance 强平数据流。
- 过滤掉微小的日常噪音。

### 7.2 流动性层（The True Impact - 升级至 MVP）

衡量抛压的真实物理破坏力。抛弃绝对金额猜测，使用相对比例。

- **输入**：`liq_notional_window`（强平金额） 与 强平发生前 100ms 的 `Bid Orderbook Depth`（例如买 1 到买 10 的挂单总额）。
- **指标**：`Impact Ratio = 强平金额 / 买盘深度`
- **逻辑**：只有当 `Impact Ratio > 预设阈值` 时，才认为真正击穿了流动性防御。

### 7.3 基差层（The True Edge - 取代原 VWAP 偏离层）

衡量理论上的修复空间（利润来源）。抛弃被强平污染的 VWAP。

- **输入**：永续合约最新中间价 `Perp_Mid_Price`，现货指数价格 `Index_Price`。
- **指标**：`Basis_Bps = (Perp_Mid_Price - Index_Price) / Index_Price * 10000`
- **逻辑**：只有当 `Basis_Bps < 负基差阈值`（例如 -50 bps）时，才证明永续被“砸出了坑”，套利者有动力进场。

### 7.4 微观衰减层（The Entry Timing - 取代原简单窗口比大小）

防范在多重瀑布中接飞刀。交易所强平是脉冲式的，仅靠“当前秒数强平金额 < 上一秒”是危险的。

- **输入**：实时成交流（Taker Buy / Taker Sell）。
- **指标**：`Order_Flow_Imbalance` 或 `Taker Buy 占比`。
- **逻辑**：不只要看强平引擎“暂时停歇”，必须看到**主动吃单买入（Taker Buy）**开始出现，证明套利资金或抄底资金正在入场抵抗，此时跟进入场才是右侧确认的左侧建仓。

## 8. 入场逻辑

### 8.1 主入场模式：被动限价单

1. 计算理论挂单价（基于 `Index_Price * (1 - 目标基差回归空间)`）。
2. 以 `Post-Only LIMIT` 方式挂出买单。
3. 为该订单设置极短有效期（如 10 秒），超时未成交说明没接到针尖，立即撤单，绝不恋战。

### 8.2 多级建仓（低延迟专属）

当端到端延迟 < 50ms 时：
1. 信号触发后，根据 `Impact Ratio` 估算潜在极值，分 2-3 级挂单。
2. 越深的单子仓位越重。

## 9. 出场逻辑

出场逻辑必须围绕“基差收敛”进行，而不是绝对价格。

### 9.1 基差回归止盈

- **目标**：永续价格向 `Index_Price` 回归。
- **触发**：当 `Basis_Bps` 回归到 0 附近或预设的正常收敛区间时止盈退出。此时套利者动力消失，多拿无益。

### 9.2 时间止损

- **逻辑**：基差套利机器人的反应是毫秒到秒级的。如果成交后 60-120 秒基差依然没有收敛，说明存在未知的结构性风险（如现货准备跟着崩），立即退出。

### 9.3 价格/基差硬止损

- **逻辑**：如果入场后，负基差不仅没收敛反而继续扩大超过 `hard_stop_bps`，无条件主动市价平仓（接受 Taker 成本）。

## 10. 初版限制

MVP 明确禁止：
- 追价买入
- 无限制补仓
- **使用任何移动平均线 (MA/EMA/VWAP) 作为交易信号或基准价**
- 忽略盘口深度的盲目抄底

## 11. 参数体系

以下是重构后基于微观结构的参数体系：

| 参数名 | 含义 | 用途 |
| --- | --- | --- |
| `liq_notional_threshold` | 强平触发绝对阈值 | 过滤小散户爆仓噪音 |
| `impact_ratio_min` | 最小冲击比例 | 强平量 / Bid深度。必须 > X 才算击穿盘口 |
| `basis_threshold_bps` | 基差偏离阈值 | 永续低于现货的 bps，决定是否有套利价值 |
| `taker_buy_confirmation` | 主动买盘确认 | 需观测到连续 X 笔或 Y 金额的 Taker Buy 才算衰减 |
| `entry_offset_bps` | 挂单偏移 | 在当前价格下方再挂的深度 |
| `order_ttl_sec` | 挂单有效期 | 错过针尖即撤 |
| `basis_tp_bps` | 基差止盈目标 | 当基差收敛至此值时退出 |
| `hard_stop_bps` | 硬止损阈值 | 负向偏移达到此值止损 |

## 12. 已冻结决策

### 12.1 彻底废弃历史成交均价 (VWAP/EMA)
在剧烈的强平事件中，价格巨幅波动伴随天量成交会严重污染 VWAP。
**新决议**：全面采用**现货指数价格 (Index Price)** 作为锚定公允价值，采用**期现基差 (Basis)** 代替绝对偏离率。

### 12.2 盘口流动性升至 P0
不再盲猜“多少万美元的强平能砸出多少坑”。
**新决议**：系统必须监听 Orderbook 增量或快照，计算 `Impact Ratio`。没有深度的对照，强平金额毫无意义。

### 12.3 止损与退出
- 止盈必须是基差收敛导向。
- 止损必须预留真实极端的滑点成本预算。
