# 术语表

## 1. 文档目的

本文档统一项目中的关键术语、时间语义和状态语义。

本文件回答的是：

- 常用术语具体指什么
- 哪些字段是交易所时间，哪些字段是本地处理时间
- 什么情况下 run 可以被视为可信

## 2. 核心术语

### 2.1 第一性原理

本项目中的“第一性原理”不是泛指“从零思考”，而是指对策略成立条件的最小可证伪因果链定义。

主定义以 [策略文档](strategy.md) 中“第一性原理定义与推导链”一节为准。

### 2.2 Liquidation Event

单笔强平事件。

最小字段应包含：

- 强平方向
- 强平价格
- 强平数量
- 强平名义金额
- `event_ts`
- `recv_ts`

### 2.3 Liquidation Episode

由多笔连续、同方向且时间上相邻的强平事件聚合而成的研究样本单位。

项目中的统计、outcome 和可交易性研究，默认都以 `episode` 为样本单位，而不是单笔强平。

### 2.4 Outcome

episode 结束后，在预设观察窗口内回填的结果记录。

它至少回答：

- 是否发生回弹
- 回弹延迟多久发生
- 回弹深度多大
- MAE / MFE 如何
- basis 或盘口恢复是否出现

### 2.5 Signal

满足当前策略准入条件后生成的一条策略判断结果。

`signal` 不等于成交，也不等于 edge 已被验证；它只是“满足当前入场条件”的结构化判断。

### 2.6 Edge

可交易 edge 指的不只是“价格后来回弹了”，而是：

- 错价真实存在
- 订单有现实可成交路径
- 计入手续费、滑点、延迟和止损成本后仍有正期望

## 3. 时间语义

### 3.1 `event_ts`

交易所事件时间。

它用于定义市场上“这件事何时发生”，是特征计算、episode 聚合和研究分析的主时间轴。

### 3.2 `recv_ts`

本地接收时间。

它用于衡量链路延迟、乱序、断流和系统处理时延，不应用来替代 `event_ts` 解释市场结构。

### 3.3 Freshness

某个价格或盘口快照与目标 `event_ts` 的时间接近程度。

若快照不满足 freshness 条件，则该快照不能参与偏离率或其他研究特征计算。

## 4. 运行模式

### 4.1 Observe Mode

只采集、聚合、研究，不下真实订单。

### 4.2 Shadow Mode

基于实时数据做虚拟挂单、虚拟成交和净收益回放，不触发真实下单。

### 4.3 Testnet Mode

在交易所测试网走真实订单生命周期，但不承受真实资金风险。

### 4.4 Live Mode

使用真实资金执行下单和风控。

## 5. 系统状态语义

主运行状态与研究可用性状态是两套正交概念。

主运行状态回答“系统当前在做什么”，研究可用性状态回答“当前 run 是否还能作为可信研究基线”。

### 5.1 `IDLE`

系统已启动但尚未进入正常观察或交易流程。

### 5.2 `WATCHING`

系统处于正常观察或待机运行状态。

在 `Observe Mode` 下，这通常表示系统正在接收数据、计算特征并输出研究结果；在可交易模式下，它表示系统允许继续评估新机会。

### 5.3 `COOLDOWN`

系统因触发阶段性风险约束而进入临时冷静期。

进入 `COOLDOWN` 后，不允许继续新开仓或接受新的订单机会，但系统仍可继续保留监控、日志和必要对账能力。

### 5.4 `HALTED`

系统已停止推进新动作，等待人工评估或恢复。

### 5.5 `OK`

当前未发现会直接污染研究结论的数据完整性故障。

### 5.6 `DEGRADED`

当前 run 发生了真实样本损坏、数据隔离、持续丢样本或等价级别的问题。

进入 `DEGRADED` 后，该 run 不能被直接当作可信研究基线。

## 6. Run 质量

### 6.1 可信 Run

满足以下条件的 run 才能作为研究基线：

- 样本无静默丢失
- 关键特征按 `event_ts` 对齐
- `episode` 定义稳定
- outcome 和可交易口径计算完整

### 6.2 污染 Run

若 run 存在样本丢失、时间错位、样本重复计数或无法区分真实可交易性，则应视为污染 run。

污染 run 可以保留作排障证据，但不能直接支持参数冻结或实盘推进。
