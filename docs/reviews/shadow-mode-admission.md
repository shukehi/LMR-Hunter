# Shadow Mode 准入条件（预注册文档）

## 重要声明

**本文档写于 2026-03-29，早于任何合格性过滤后的样本统计分析。**

预注册（pre-registration）的目的是：在看到充分数据之前，先冻结进入 Shadow Mode
的判断规则。如果在积累数据之后才定义准入条件，条件会不可避免地与已有数据匹配，
从而产生后验偏差（hindsight bias），使得"准入"丧失客观意义。

本文档一经提交，**不允许以"数据与预期不符"为理由修改阈值**。
唯一合法的修改原因：发现技术性错误（如计算公式有误、字段语义变更）。

---

## 1. Shadow Mode 的目标定义

Shadow Mode 的目的不是"验证策略盈利"，而是：

1. **验证执行可行性**：限价单在真实订单簿中的实际成交率，是否接近模拟成交率
2. **验证策略机制**：止损、止盈、时间止损的触发逻辑在实盘中是否按预期工作
3. **建立执行摩擦基准**：量化模拟结果与真实执行结果之间的系统性偏差

Shadow Mode **不是**参数优化的场所，进入时参数必须提前固定。

## 2. Episode 合格性判定标准（单条样本）

以下所有条件必须同时满足，一条 episode 才计入有效研究样本。

### 2.1 数据完整性（必要条件）

| 检查项 | 判断标准 | 字段来源 |
|--------|---------|---------|
| 运行可信 | `overflow_dropped = 0 AND isolated = 0 AND lost = 0` | `service_heartbeats.notes` |
| MAE 可用 | `mae_bps IS NOT NULL` | `episode_outcomes.mae_bps` |
| MFE 可用 | `mfe_bps IS NOT NULL` | `episode_outcomes.mfe_bps` |
| 时间止损价格可用 | `price_at_5m IS NOT NULL` | `episode_outcomes.price_at_5m` |
| 结果窗口数据密度 | `trade_count_0_15m >= 30` | `episode_outcomes.trade_count_0_15m` |
| 入场基准价可用 | `entry_price IS NOT NULL` | `episode_outcomes.entry_price` |
| 事前基准可用 | `pre_event_index_price IS NOT NULL` | `liquidation_episodes.pre_event_index_price` |

### 2.2 信号真实性（必要条件）

| 检查项 | 判断标准 | 说明 |
|--------|---------|------|
| 最低清算名义价值 | `liq_notional_total >= 100,000 USD` | 低于此值的清算对盘口冲击可忽略不计 |
| 最低基差深度 | `max_basis_bps <= -10` （注：max_basis_bps 为负值） | 基差过浅无法覆盖套利和滑点成本 |
| 最低强平笔数 | `liq_count >= 3` | 单笔或双笔不构成级联踩踏结构 |

### 2.3 因果结构有效性（必要条件）

| 检查项 | 判断标准 | 说明 |
|--------|---------|------|
| 反弹目标距离有意义 | `basis_rebound_depth >= 10` | 低于此值，止盈触发后净收益几乎被手续费完全吞噬 |
| 方向正确 | `side = 'SELL'` | 策略仅研究 SELL 踩踏做多反弹 |

### 2.4 统计独立性（必要条件）

| 检查项 | 判断标准 | 说明 |
|--------|---------|------|
| 与上一条合格 episode 间隔 | `start_event_ts - prev_qualified_end_ts >= 900,000ms`（15 分钟） | 确保 outcome 观测窗口不重叠 |

**注**：独立性判断基于"合格 episode"序列，不包含被过滤掉的 episode。

## 3. 最低样本数量要求

进入 Shadow Mode 需满足以下全部数量条件：

| 条件 | 最低要求 | 说明 |
|------|---------|------|
| 合格样本总数 | ≥ 60 条 | 基本参数对比的统计下限 |
| 美盘合格样本 | ≥ 25 条 | 主力时段，要求最高 |
| 欧盘合格样本 | ≥ 15 条 | 次要时段 |
| 亚盘合格样本 | ≥ 10 条 | 低流动性时段，可接受较少样本 |
| 数据跨越时间 | ≥ 30 个自然日 | 确保覆盖多个市场状态（趋势/震荡/高波/低波） |
| 独立 run 数 | ≥ 3 次（每次 run 为进程重启后的连续运行） | 避免单次运行中的系统性偏差 |

---

## 4. 统计准入条件

样本数量达标后，需通过以下统计条件。

### 4.1 必要条件（全部满足才可进入）

**条件 A：全局期望值方向合理**
```
在默认参数（entry_offset=0, hard_stop=30bps）下：
期望净收益点估计 > -10 bps

（宽松设定：Shadow Mode 本身用于验证执行，不要求 Observe Mode 已证明盈利）
```

**条件 B：至少一个参数组合在主力时段（美盘或欧盘）显示正期望**
```
sensitivity_table 中：
存在至少一个 (entry_offset, hard_stop) 组合，使得：
  - 美盘或欧盘的期望净收益 > 0 bps
  - 对应成交率 >= 20%（避免成交率过低的样本外结论）
```

**条件 C：无明显负边际时段**
```
任一时段的期望净收益 > -30 bps
（期望值 < -30 bps 说明该时段在模拟层面就已经严重亏损，不应进入实盘验证）
```

### 4.2 参数鲁棒性条件（必须满足）

**条件 D：正期望参数区域至少跨越 2 个 hard_stop 值**
```
如果某个 entry_offset 在 sensitivity_table 中显示正期望，
则该正期望必须在至少 2 个不同的 hard_stop 值下成立。

理由：只在单一 hard_stop 下为正的期望极可能是过拟合。
```

### 4.3 禁止进入的明确场景（任一触发则禁止）

- 亚盘合格样本 < 10 条，且亚盘期望值 < -20 bps（数据不足 + 负信号，不可盲目交易）
- 数据跨越时间 < 30 天（单一市场状态的数据不具备外推能力）
- 最近 7 天内有任何 CONTAMINATED run 记录（数据可信性问题）

## 5. Shadow Mode 参数选择规则

参数必须在满足准入条件后、**开始 Shadow Mode 运行前**一次性确定，并写入配置文件冻结。

### 5.1 参数选择方法

**禁止**：选择 sensitivity_table 中期望值最高的单点参数。

**要求**：选择"鲁棒区间"——在 sensitivity_table 中，连续多个格子期望值均为正的区域的**中间值**。

```
鲁棒区间识别方法：
1. 筛选出所有期望值 > 0 且成交率 >= 20% 的格子
2. 找到连续相邻格子构成的最大矩形区域
3. 选择该区域的中心点参数
4. 如果没有连续区域（只有孤立格子），不允许进入 Shadow Mode
```

### 5.2 初始 Shadow Mode 配置约束

| 参数 | 约束 | 说明 |
|------|------|------|
| entry_offset_bps | 依照 5.1 规则确定，不得手动调整 | — |
| hard_stop_bps | 依照 5.1 规则确定，不得手动调整 | — |
| time_stop_sec | 固定为 300 秒（5 分钟） | 与 Observe Mode 模拟一致 |
| 单笔名义仓位 | ≤ $500 USD | Shadow Mode 目的是验证机制，不是赚钱 |
| 同时持仓数 | ≤ 1 | 避免多个 episode 同时入场的复合风险 |
| 最大日亏损 | ≤ $50 USD | 触发则当日停止，次日重启 |

### 5.3 参数锁定期

Shadow Mode 启动后，**至少完成 20 笔实际成交的交易（非 episode，是实际下单成交）**，
才允许讨论参数调整。20 笔成交前的任何"调参冲动"都视为对小样本噪声的反应。

## 6. Observe Mode 的已知局限性声明

以下局限性在进入 Shadow Mode 前必须明确认知，不得以"后续解决"为由绕过：

### 局限 1：入场价系统性乐观偏差

```
当前模拟使用 entry_price = min_mid_price（episode 期间最低价）
真实执行时，最低价发生在 episode 进行中，只能在 episode 结束后才能下单
```

**影响**：真实成交率 < 模拟成交率；真实期望值 < 模拟期望值（幅度未知）

**缓解措施**：Shadow Mode 启动后，将真实成交率与模拟成交率做对比，量化偏差。

### 局限 2：时间止损价格为快照，非实际成交价

```
模拟使用 price_at_5m（5 分钟成交序列的最后一笔价格）作为时间止损价
真实执行时存在滑点，实际成交价可能低于此快照价
```

**影响**：时间止损的实际亏损略大于模拟亏损。

### 局限 3：样本量不足以支撑参数优化

```
当前方案基于 <60 条合格样本
sensitivity_table 中的"最优参数"大概率是过拟合
```

**缓解措施**：依照 5.1 的鲁棒区间原则选参，不选单点最优。

## 7. 准入流程

```
阶段 1（当前）
  └── 积累 Observe Mode 合格样本
  └── 同步实现 price_at_episode_end 字段（修复入场价偏差）
  └── 实现 episode 合格性过滤器（qualify_episode）

阶段 2（样本达标后）
  └── 运行 qualify_episode 过滤
  └── 确认满足第 3 节数量要求
  └── 运行 research_report，检查第 4 节统计条件
  └── 如条件满足 → 执行第 5 节参数选择 → 写入配置并冻结
  └── 如条件不满足 → 继续积累，不进入 Shadow Mode

阶段 3（Shadow Mode 运行中）
  └── 每完成 5 笔实际成交，对比真实 vs 模拟的成交率和期望值
  └── 完成 20 笔成交后，评估是否需要参数调整
  └── 累计亏损超过 $200 USD → 暂停 Shadow Mode，重新审查
```
